部署帮助中心到 Cloudflare Pages

帮助中心建议放到独立公开仓库，然后用 Cloudflare Pages 自动部署，并绑定 help.c2ly.com。

推荐结构

源码私有仓库：只放系统代码，不公开。
帮助中心公开仓库：只放教程，不放源码、不放客户资料、不放密钥。
Cloudflare Pages：从帮助中心公开仓库自动部署。

GitHub 仓库

1. 新建公开仓库，例如 c2ly-help。
2. 把 help-site/ 目录里的文件上传到仓库根目录。
3. 确认仓库根目录有 index.html、_redirects、assets/help.css。

Cloudflare Pages 设置

1. 登录 Cloudflare Dashboard。
2. 进入 Workers & Pages。
3. 点击 Create application。
4. 选择 Pages，连接 GitHub 仓库。
5. 选择 c2ly-help 仓库。
6. Framework preset 选择 None。
7. Build command 留空。
8. Output directory 填 / 或留空，按 Cloudflare 当前界面要求设置。
9. 点击 Deploy。

绑定 help.c2ly.com

1. 部署成功后，进入 Pages 项目。
2. 打开 Custom domains。
3. 添加 help.c2ly.com。
4. 如果 c2ly.com DNS 已托管在 Cloudflare，系统会自动添加 CNAME。
5. 等待证书变成 Active。

后台链接

系统后台帮助链接默认指向 https://help.c2ly.com/...。
如果以后换域名，在后台 Plateforme commerciale 里修改教程 URL 即可。

禁止上传

不要上传系统源码。
不要上传客户数据、数据库备份、订单信息、发票、密钥、服务器配置。
截图里如果出现 token、手机号、客户姓名，必须打码后再放入教程。
